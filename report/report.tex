\begin{abstract}
We present an end-to-end optical Sudoku solver that combines classical computer vision techniques with convolutional neural network (CNN) based digit recognition. The system processes an input image of a Sudoku puzzle, detects and rectifies the grid using perspective transformation, extracts individual cells, and classifies digits using a CNN trained on the Chars74K printed-digit dataset. To improve robustness, the system includes orientation invariance, adaptive thresholding, and cell normalization tuned to suppress residual grid borders while preserving fragile digit structure. Experimental results show that digit recognition is no longer the dominant failure mode of the pipeline; instead, the remaining errors are primarily caused by grid detection and downstream cell extraction quality.
Source code is available at \url{https://github.com/eddie-an/Optical-Sudoku-Solver}.
\end{abstract}

\section{Introduction}
Sudoku is a widely recognized constraint-based puzzle that has become a common benchmark problem for evaluating computer vision pipelines involving detection, segmentation, and classification. Automatically solving Sudoku from images presents several challenges, including perspective distortion, noise, lighting variation, and digit recognition.

In this work, we develop a complete optical Sudoku solver that integrates image preprocessing, geometric transformation, and machine learning. Unlike end-to-end deep learning approaches, our method emphasizes interpretability and modularity by retaining a classical computer vision pipeline for board detection while using a learned model only where it is most valuable: digit recognition.
Custom functions are implemented for preprocessing steps such as Gaussian filtering, adaptive thresholding, polygon approximation, perspective transformation, and cell extraction, while leveraging OpenCV \cite{bradski2000opencv} for contour detection. The final digit recognition component is implemented using a convolutional neural network (CNN) trained on the Chars74K printed-digit dataset \cite{deCampos09}. We initially explored an SVM using handcrafted Histogram of Oriented Gradients (HOG) \cite{dalal2005histograms} and Local Binary Patterns (LBP) \cite{ojala2002multiresolution} features, but the feature-based approach proved too sensitive to malformed crops, residual grid borders, and disconnected digit strokes. Replacing the SVM with a CNN, and then improving the normalization pipeline that fed it, substantially improved recognition robustness on real Sudoku cells.

\section{Methodology}

\subsection{Initial Grid Detection Strategy}
Initially, we explored using Harris Corner Detection \cite{harris1988combined} to directly identify the intersections of the grid lines and locate the four outer corners of the board. However, this approach proved to be highly sensitive to noise, text within the cells, and background textures. The resulting corner response yielded numerous false positives, making it difficult to reliably isolate the main corners of the Sudoku board as shown in Figure \ref{fig:harris_corner}.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.45\textwidth]{images/harris_corner_response.png}
    \caption{Harris corner detection response. The high number of false positives from text and noise made reliable grid extraction unfeasible.}
    \label{fig:harris_corner}
\end{figure}

Due to this unreliability, we pivoted to an edge-based topological approach, utilizing contour detection paired with a polygon approximation algorithm, which provided a much more robust extraction of the grid's outer boundary.

\subsection{Preprocessing and Grid Detection}
The input image is first converted to grayscale and smoothed using a Gaussian filter to reduce noise. We implement Gaussian filtering manually using a generated kernel and convolution operation.

Adaptive thresholding is applied to convert the image into a binary representation. Unlike global thresholding, adaptive thresholding computes a local threshold for each pixel, allowing the system to handle uneven lighting conditions.
Here is a comparison between global and adaptive thresholding (see Figure \ref{fig:thresholding}):
\begin{figure}[ht]
    \centering
    \includegraphics[width=0.22\textwidth]{images/otsu_thresholding.png}
    \includegraphics[width=0.22\textwidth]{images/adaptive_thresholding.png}
    \caption{Comparison of Otsu global thresholding (left) and adaptive thresholding (right).
    Adaptive thresholding provides better results under non-uniform lighting conditions.}
    \label{fig:thresholding}
\end{figure}

The resulting binary image is then processed to detect contours.

Contours are extracted using OpenCV's \texttt{findContours} function, which implements the topological structural analysis developed by Suzuki and Abe \cite{suzuki1985topological}. This function returns a list of contours found in the binary image.
Each contour is given in the format of \texttt{[[x\_1, y\_1], [x\_2, y\_2], ..., [x\_n, y\_n]]}, where $(x_i, y_i)$ are the coordinates of the contour points. The area of each contour is calculated using the Shoelace Formula, which computes the area based on the vertices of the polygon formed by the contour points. 
The contour with the largest area that approximates a quadrilateral is assumed to correspond to the Sudoku grid (see Figure \ref{fig:contour_detection}).


\begin{figure}[ht]
    \centering
    \includegraphics[width=0.45\textwidth]{images/contours.png}
    \caption{Contour detection result showing the three largest contours by area. The green outline represents the largest quadrilateral contour identified as the Sudoku grid.}
    \label{fig:contour_detection}
\end{figure}


To identify the corners of the grid, we apply a polygon approximation algorithm (Ramer-Douglas-Peucker \cite{douglas1973algorithms}) to simplify the contour from n points to four points.
These four corner points are then used for perspective transformation to obtain a top-down view of the Sudoku board, which is essential for accurate cell extraction and digit recognition.


\subsection{Perspective Transformation}
Once the grid corners are detected, a perspective transformation is applied to obtain a top-down view of the Sudoku board. The board is warped into a fixed square resolution (450x450), enabling consistent downstream processing.
The perspective transformation is computed using the four detected corner points and the corresponding points in the output image. The transformation matrix is calculated using the \texttt{get\_perspective\_transform} function, and the image is warped using the \texttt{warp\_perspective\_inverse} function. These functions are custom implementations of OpenCV's perspective transformation, which we implemented manually to better understand the underlying mathematics and to allow for more control over the warping process.

One caveat to perspective transformation is that it can create holes in the output image if forward mapping is used, as some pixels in the output may not correspond to any pixel in the input. Forward warping maps each pixel from the input to the output, which can lead to gaps if multiple input pixels map to the same output pixel or if some output pixels do not receive any input mapping. In contrast, inverse warping maps each pixel in the output back to the input, ensuring that every pixel in the output is assigned a value based on the corresponding location in the input image, thus avoiding holes. 
Since the perspective transformation process can result in pixel coordinates having decimal values attached to them, these methods also rely on an interpolation method to get integer coordinates. We tested three interpolation methods:
\begin{enumerate}
    \item \textbf{Floor Truncation:} Simply taking the integer value of the calculated coordinates, effectively performing a floor operation ($\lfloor x \rfloor, \lfloor y \rfloor$).
    \item \textbf{Rounding:} Mapping to the nearest integer coordinate.
    \item \textbf{Bilinear Interpolation:} Computing a weighted average of the four nearest pixels.
\end{enumerate}

\begin{quote}
\small
While the first two methods were tested with both forward and inverse mapping, bilinear interpolation was only implemented with inverse mapping. Implementing bilinear interpolation for forward warping is significantly more complex because there are cases where multiple pixels from the original image could get mapped to the same pixel in the new one. 
\end{quote}
As seen in Figure \ref{fig:perspective_transform} below, we decided that inverse warping using bilinear interpolation resulted in the best visual quality and incldued it in the pipeline.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.22\textwidth]{images/forward_int.png}
    \includegraphics[width=0.22\textwidth]{images/inverse_int.png}
    \includegraphics[width=0.22\textwidth]{images/forward_round.png}
    \includegraphics[width=0.22\textwidth]{images/inverse_round.png}
    \includegraphics[width=0.22\textwidth]{images/inverse_bilinear.png}
    \caption{Perspective transformation results showing the forward warped image (left column) and the inverse warped top-down view of the Sudoku board (right column) with different interpolation methods (from top row to bottom row: integer value, rounding, and bilinear).}
    \label{fig:perspective_transform}
\end{figure}

\subsection{Cell Extraction}
The warped square board is divided into a $9 \times 9$ grid, producing 81 individual cells. Each cell is cropped with margins to remove grid lines (see Figure \ref{fig:cell_extraction}).

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.22\textwidth]{images/top_left_cell.png}
    \includegraphics[width=0.22\textwidth]{images/top_right_cell.png}
    \caption{Example of extracted cells from the warped Sudoku board. The left image shows the top-left cell, while the right image shows the top-right cell. Margins are applied to avoid grid lines.}
    \label{fig:cell_extraction}
\end{figure}

To determine whether a cell is empty, we compute the ratio of foreground (ink) pixels using adaptive thresholding. Cells with low ink density are classified as empty and skipped during digit recognition.

\subsection{Orientation Invariance}
To handle images captured at arbitrary rotations, the system evaluates four orientations (0°, 90°, 180°, 270°). For each orientation, digit classification confidence scores are computed from the softmax outputs of the trained CNN.
The orientations are sorted in descending order of average confidence.
The system attempts to solve the Sudoku puzzle starting with the highest confidence orientation. If a valid solution is found, it is returned; otherwise, the next orientation is evaluated until a solution is obtained or all orientations are exhausted.

\subsection{Digit Recognition}
Digit classification is performed using a convolutional neural network. Each extracted cell is first normalized to a fixed $40 \times 40$ canvas so that the digit is centered and scaled consistently. During development, this normalization stage became a major focus: several failure cases were caused not by the classifier itself, but by residual cell borders and fragmented digit (see Figure \ref{fig:digit_normalization}). To address this, the normalization pipeline was refined to explicitly remove dense border-aligned line fragments and to preserve multiple meaningful connected components when needed, which was particularly important for printed digit 4.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.45\textwidth]{images/digit_normalization.png}
    \caption{Normalization stages for two representative difficult cells. The raw extracted cells are first thresholded, then cleaned to suppress border-aligned artifacts and other contamination, before being centered and scaled for CNN classification.}
    \label{fig:digit_normalization}
\end{figure}

The final CNN uses four convolutional blocks with batch normalization, ReLU activations, max pooling, and dropout, followed by a lightweight fully connected classifier. This design was chosen to remain compact enough for project inference while still learning spatial structure directly from normalized cell images. Compared with the earlier SVM pipeline, the CNN has two key advantages:
\begin{itemize}
    \item \textbf{It learns features from data directly,} rather than relying on fixed handcrafted descriptors such as HOG \cite{dalal2005histograms} and LBP \cite{ojala2002multiresolution}.
    \item \textbf{It is more tolerant of small geometric and topological distortions,} such as stroke thickening, disconnected components, and minor border remnants.
\end{itemize}

The model was trained on the Chars74K printed-digit dataset rather than on cells mined from the Sudoku dataset. We initially attempted to use extracted Sudoku cells for training, but the quality of automatic grid detection and cell extraction was too inconsistent; noisy training labels from misdetected boards would have degraded the recognizer rather than improved it. Chars74K provided a cleaner supervised training source, while the Sudoku dataset \cite{wichtSudoku} was reserved for evaluation on real pipeline outputs.

We also explored an SVM based on HOG and LBP features and a transfer-learning variant using MNIST pretraining before Chars74K finetuning. In practice, neither alternative removed the main failure modes. The largest gains came from switching to the CNN and improving normalization so that recognizable digits were preserved before classification.

\subsection{Sudoku Solving}
The digits recognized from the cells are used to populate a partially filled Sudoku board.
This board is validated and solved using a backtracking algorithm. The final solution is displayed if a valid configuration is found (see Figure \ref{fig:sudoku_solution}).

\section{Results}
The proposed optical Sudoku solver demonstrated highly reliable performance under ideal to moderate real-world conditions. The integration of adaptive thresholding and inverse perspective warping proved highly effective in standardizing the grid against non-uniform lighting and keystoned (slanted) viewing angles. Furthermore, the orientation-invariant classification mechanism successfully resolved puzzles captured at arbitrary rotations.

The final digit recognizer was evaluated on a subset of Sudoku dataset images for which the extracted boards and cells were judged suitable for recognition (see Figure \ref{fig:cnn_confusion_matrix}). On this subset, the CNN achieved 98\% accuracy over 2374 labeled digit cells, with class-wise recall of at least 96\% for every digit and 98\% recall for digit 4 after the normalization improvements. This represents a clear improvement over the earlier SVM-based approach, particularly on difficult cells containing minor distortions or residual border artifacts.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.45\textwidth]{images/CNN_confusion_matrix.png}
    \caption{Confusion matrix of the final CNN digit recognizer on the evaluated Sudoku-cell subset. Most digits lie strongly on the diagonal, indicating that recognition errors are comparatively rare once the extracted cell is of reasonable quality.}
    \label{fig:cnn_confusion_matrix}
\end{figure}

These experiments also helped clarify the main limitation of the end-to-end system. After the CNN and normalization pipeline were refined, digit recognition became highly reliable on well-extracted cells. The remaining errors were more often associated with earlier stages of the pipeline, such as grid detection, perspective rectification, and cell extraction quality as shown in Figure \ref{fig:cnn_worst_errors}. This suggests that further end-to-end improvements will depend more on strengthening board localization and cell extraction than on increasing classifier complexity.

\begin{figure}[ht]
    \centering

Many of the remaining failure cases are associated with malformed or contaminated inputs rather than clean, well-centered digits, reinforcing that the dominant bottleneck has shifted upstream to grid detection and cell extraction quality.}

\section{Discussion}
While the proposed system performs reliably end-to-end under ideal conditions, real-world testing revealed several distinct limitations and edge cases:
\begin{itemize}
\item \textbf{Information Loss in Blurry Images:} The Suzuki-Abe topological structural analysis algorithm (used by OpenCV's contour detection) relies heavily on continuous binary lines. Blurry images suffer from information loss, leading to fragmented lines after thresholding. We attempted to mitigate this by calculating the variance of the Laplacian to estimate a blur score, bypassing the Gaussian blur step if the score fell below a certain threshold. We also experimented with morphological operations (closing and dilation) to reconnect broken lines. However, while this fixed some blurry cases, it degraded the performance on clear images, making it an unreliable global solution.

\item \textbf{Partial Grid Visibility:} Because the contour detection requires a closed boundary to identify the board, the algorithm fails if the outer edge of the Sudoku grid is cut off from the image frame. An unbroken quadrilateral cannot be formed, causing the pipeline to fail early (see Figure \ref{fig:disconnected_lines}).

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.22\textwidth]{images/original_3.jpg}
    \includegraphics[width=0.22\textwidth]{images/contour_fail_2.png}
    \caption{Failure case due to partial grid visibility. The left edge is cut off from the image frame, preventing closed contour detection. The right image shows the failed contour detection result, where no valid quadrilateral is identified.}
    \label{fig:disconnected_lines}
\end{figure}

\item\textbf{Background Contour Interference:} The pipeline currently operates on the assumption that the Sudoku board will be the largest contour in the field of view. If a larger, closed geometric shape is present in the background, the algorithm will incorrectly target it instead of the puzzle (see Figure \ref{fig:non_max_contour}). This issue is relatively minor in practice, as the user can easily adjust the camera distance to ensure the board dominates the frame.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.22\textwidth]{images/original_2.png}
    \includegraphics[width=0.22\textwidth]{images/contour_fail.png}
    \caption{Failure case due to background contour interference. A larger shape in the background is incorrectly identified as the Sudoku board (see green outline on the right image).}
    \label{fig:non_max_contour}
\end{figure}

\item \textbf{Non-linear Distortions:} Perspective transformation assumes the original surface is planar. If the paper itself is bent or curved excessively, warping the four outer corners to a fixed $450 \times 450$ square does not straighten the inner grid lines. Consequently, our assumption that each cell is geometrically uniform fails; curved grid lines encroach into the extracted cell margins. The \texttt{is\_cell\_empty} function detects this grid ink, falsely classifying the cell as populated, and the normalization stage may still receive malformed inputs that do not correspond to a clean digit.

\item \textbf{Classifier Bottleneck Removed:} One of the main outcomes of the final iteration was identifying that digit recognition was not the limiting component once the CNN replaced the SVM. The initial SVM was constrained by handcrafted features: HOG and LBP worked well on clean centered digits, but performance degraded quickly when the cell contained disconnected strokes, clipped shapes, or residual border lines. The CNN was substantially more robust to these effects, especially after targeted normalization changes. As a result, the practical bottleneck shifted upstream to grid detection and cell extraction.
\end{itemize}
Despite these limitations, the modular design of the pipeline allows individual components to be improved independently. Our final system already benefits from replacing the SVM with a CNN for digit recognition, and the next meaningful improvement would be implementing a more geometry-aware board extraction strategy, such as grid-line intersection mapping, to better handle bent or non-linear grids.


\section{Conclusion}
In this paper, we presented a complete, modular optical Sudoku solver combining classical computer vision pipelines with learned digit recognition. By utilizing contour detection and polygon approximation, we successfully extracted Sudoku grids from unstructured images. Applying inverse perspective warping allowed us to geometrically normalize puzzle cells and feed them to a CNN trained on Chars74K printed digits.

While the system effectively navigates common challenges like perspective distortion and lighting variations, our testing highlighted the limitations of classical topological algorithms when faced with fragmented lines, partial framing, and non-planar physical surfaces. Demonstrating the viability of hybrid approaches for structured visual tasks, this work also underscores the necessity for resilient error-handling in early preprocessing stages.

Future work will therefore focus primarily on improving grid detection and cell extraction, especially for warped pages, partial framing, and cases where residual borders contaminate the extracted cells. A more robust geometric model of the Sudoku lattice would likely produce larger end-to-end gains than further increasing classifier complexity.
